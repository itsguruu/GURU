# Security Policy

## Supported Versions

We actively provide security updates for the following versions of **GURU MD**:

| Version   | Supported          | Security Updates Until     | Notes                              |
|-----------|--------------------|----------------------------|------------------------------------|
| 4.5.x     | ✅ Yes             | At least until Dec 2026    | Current stable & recommended       |
| 4.4.x     | ❌ No              | Expired                    | Upgrade recommended                |
| 4.0.x – 4.3.x | ❌ No          | Expired                    | Upgrade to 4.5.x required          |
| < 4.0     | ❌ No              | Not supported              | Major security & compatibility issues |

**Note:** Only the latest minor version in the 4.5.x series receives **active security patches**.  
We strongly recommend all users stay on the latest release.

## Reporting a Vulnerability

**We take the security of GURU MD very seriously.**  
If you discover a security vulnerability, **please do NOT open a public issue** — report it privately so we can fix it before it becomes public knowledge.

### How to Report

Please send your report to one of the following trusted contacts:

- **Primary contact (project owner):**  
  Email: [guru.tech.official@gmail.com](mailto:guru.tech.official@gmail.com)  
  WhatsApp: +254778074353 (preferred for quick communication)

- **Alternative contact:**  
  Open a private issue on GitHub:  
  https://github.com/itsguruu/GURU/security/advisories/new

### What to Include in Your Report

To help us understand and resolve the issue quickly, please include as much of the following as possible:

- Type of vulnerability (XSS, command injection, session hijacking, etc.)
- Full description of the issue
- Steps to reproduce (with screenshots/videos if possible)
- Affected version(s)
- Proof-of-concept code or exploit (if safe to share)
- Potential impact
- Any suggested fixes/mitigations (optional)

### What to Expect After Reporting

- **Within 48 hours:** You will receive an acknowledgment of your report
- **Within 7–14 days:** We will provide an update on our investigation (accepted, declined, need more info, etc.)
- **If accepted:** We will work on a fix (timeline depends on severity)
- **Severity-based timeline:**
  - Critical: fix within 72 hours + coordinated disclosure
  - High: fix within 7–14 days
  - Medium/Low: fix in next minor release
- **If declined:** We will explain why (e.g., not reproducible, out of scope, already known)
- **Credit:** We will publicly credit you (unless you prefer to remain anonymous) in the release notes and security advisory

Thank you for helping keep **GURU MD** secure!
